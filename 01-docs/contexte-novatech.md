# Contexte métier — NovaTech Services

## Présentation de l’entreprise

NovaTech Services est une PME fictive composée d’environ 80 salariés.

L’entreprise dispose d’un service informatique interne chargé de gérer les incidents, les demandes d’assistance et le suivi du matériel informatique.

## Situation actuelle

Actuellement, les demandes informatiques sont transmises de plusieurs façons :

- par mail ;
- par téléphone ;
- oralement auprès d’un technicien ;
- parfois par message interne.

Cette organisation pose plusieurs problèmes :

- certaines demandes sont oubliées ;
- les techniciens ne disposent pas toujours d’un historique clair ;
- les priorités ne sont pas toujours identifiées ;
- les salariés ne savent pas toujours où en est leur demande ;
- le responsable informatique manque d’indicateurs sur l’activité du support.

## Problème métier

NovaTech Services souhaite centraliser les demandes informatiques dans une application unique afin d’améliorer le suivi des incidents et la qualité du support informatique.

## Solution proposée

La solution proposée est TicketFlow, une application web de gestion de tickets informatiques.

Elle permettra aux salariés de créer une demande, aux techniciens de la prendre en charge, et à l’administrateur de suivre l’ensemble des tickets.

## Objectifs du projet

Les objectifs principaux sont :

- centraliser les demandes informatiques ;
- suivre l’état d’avancement des tickets ;
- attribuer les tickets aux techniciens ;
- conserver un historique des échanges ;
- faciliter la priorisation des incidents ;
- fournir une meilleure visibilité au service informatique.

## Acteurs concernés

| Acteur | Rôle |
|---|---|
| Salarié | Crée un ticket et suit son avancement |
| Technicien | Prend en charge les tickets et met à jour leur statut |
| Administrateur | Gère les utilisateurs, attribue les tickets et supervise l’activité |
| Responsable informatique | Consulte les indicateurs et suit la qualité du support |

## Périmètre du projet

Le projet couvrira les fonctionnalités suivantes :

- authentification des utilisateurs ;
- création d’un ticket ;
- consultation des tickets ;
- changement de statut ;
- attribution d’un ticket ;
- ajout de commentaires ;
- tableau de bord simple ;
- historique des actions principales.

## Limites du projet

Dans le cadre du BTS SIO SLAM, certaines fonctionnalités resteront simulées ou simplifiées :

- pas d’envoi réel d’e-mails ;
- pas de connexion à un annuaire Active Directory ;
- pas de système de notification temps réel ;
- pas d’application mobile dédiée.
